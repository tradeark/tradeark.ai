AI 風控防火牆

讓 AI 透過本地防火牆交易，而不是直接持有原始交易所金鑰

TradeArk 內建兩層 AI 交易防火牆，面向 OpenClaw、Claude Code、Codex 等本地智能體。你可以在本地 Web UI 中建立具名 AI 用戶端，產生可直接貼上的提示詞，並讓 TradeArk 在任何交易所寫入請求離開本機前先執行限制。

按智能體劃分權杖範圍

為每個機器人、工作區或操作員單獨建立 AI 用戶端。每個權杖都綁定到選定的已儲存帳戶，並可在本地 Web UI 中即時輪換或撤銷。

交易前策略引擎

訂單到達交易所前，TradeArk 可以執行路由允許清單、交易對規則、資產範圍、槓桿上限、最大數量、最大名義金額以及頻率或每日限制。

即時看護與熔斷開關

第二層會持續透過本地執行器監控餘額和持倉。當觸發閾值時，它可以告警、撤銷掛單、平掉合約敞口，或凍結用戶端/帳戶。

可直接用於提示詞的接入流程

AI 風控面板可為 OpenClaw、Claude Code、Codex 等工具產生可直接貼上的提示詞，團隊無需手寫請求標頭、帳戶範圍或路由規則。

餘額、持倉、歷史等一般本機讀取都留在同一台機器上，而交易所寫入只需為每個請求加上已命名用戶端標記，就能在 AI 風控下走相同的本機交易路由。

操作員可以一鍵輪換權杖、凍結用戶端或帳戶，並把每個 AI 工作流程限制在已儲存帳戶範圍內，而不是發放不受限制的真實交易所金鑰。

控制平面由操作員掌握。AI 取得可直接使用的提示詞存取方式，TradeArk 在本地評估每個受保護寫入請求，交易所只會看到通過策略的請求。

操作員 -> AI prompt -> 本地讀取 -> 受保護寫入 -> 策略閘道 -> Exchange -> 看護器
1
建立具名 AI 用戶端

在本地 Web UI 中將它綁定到已儲存帳戶、允許路由和策略限制。

2
產生 AI 可直接使用的提示詞

提示詞會告訴 AI 該呼叫哪些本機路由，以及如何為受保護的寫入給每個請求加上已命名用戶端標記。

3
透過已命名的本機用戶端執行交易所寫入

下單、全部平倉和撤銷掛單都會先進入本地防火牆，而不是用原始交易所憑證繞過控制。

4
攔截、放行或凍結

第一層可在交易所看到請求前先攔截。第二層可在風險敞口出現後告警、撤單、平倉或凍結。

安裝說明

如果你是自己安裝，請使用常規安裝方式；如果你在準備伺服器或 AI 工作流程，請使用一行安裝命令。

Windows

Portable ZIP for the local browser-based trading UI. On first open you can keep the page public or require a username and password every time. No admin rights, registry writes, or auto-start.

下載 Windows 便攜 ZIP 包 Windows x64 · 1.0.0+20260628.092211Z

將 ZIP 解壓到任何使用者可寫目錄，然後運行本地啟動器：

1. 下載並解壓 tradeark_windows_portable.zip
2. 打開解壓後的資料夾
3. 運行 Start TradeArk.cmd
4. Your browser opens the local UI and the first-run access wizard

如果你更喜歡用腳本安裝到 LocalAppData，請使用這個 PowerShell 輔助腳本：

Invoke-WebRequest -Uri https://tradeark.ai/install.ps1 -OutFile install.ps1
powershell -ExecutionPolicy Bypass -File .\install.ps1 -Yes

本地介面地址：

http://127.0.0.1:38182/

This Windows package stays in user space. It does not require admin rights, does not write startup registry entries, and only notifies about new versions. Later you can change Web UI access from Settings > Web UI Access, or reset it from the portable folder with: TradeArk.exe web-ui set-password --username <name>

macOS

Menu bar app that installs TradeArk locally and lets you open the trading page or restart the local service with one click. On first open you can leave the page public or protect it with a username and password.

下載 TradeArk for macOS (.dmg) 下載 Intel DMG macOS 13+ · Apple Silicon / Intel

對大多數 macOS 桌面使用者來說，直接使用打包好的應用即可：

1. 下載並打開 TradeArk macOS 安裝包 (.dmg)
2. 將下載的應用拖入 /Applications
3. 打開應用，然後使用選單列圖示在瀏覽器中打開本地介面
4. 透過選單重啟本地服務，或開啟背景自動啟動

這個 DMG 可以在 Finder 中正常打開，但還沒有完成 Apple notarization。第一次打開時，macOS 仍可能要求你右鍵應用並選擇一次「打開」。

如果你需要命令列部署或無人值守安裝，也可以繼續使用一行安裝命令：

curl -fsSL https://tradeark.ai/install.sh | bash -s -- --non-interactive --skip-credentials --install-service

The command-line install sets up the same local TradeArk service. The app is simply a more convenient desktop way to manage it. If you enable password mode, every browser visit asks you to sign in until you switch back to public access or reset the password from Settings > Web UI Access.

Linux (Ubuntu / Debian)

執行一條命令，Linux 會下載 TradeArk；首次執行會完成安裝，之後可原地更新既有安裝。第一次開啟時，你可以選擇保持頁面公開，或每次訪問都要求使用者名稱和密碼：

curl -fsSL https://tradeark.ai/install.sh | bash -s -- --non-interactive --skip-credentials --install-service

當 TradeArk 已安裝時，再次執行同一命令會保留本地資料，盡可能復用目前 Linux 服務設定，並自動重啟本地服務。

當 Linux 版本在本地 Web UI 中提示有新版本時，點擊右上角更新提示會自動執行原地更新，並顯示進度直到本地服務恢復。Windows 和 macOS 仍會開啟手動下載/更新連結。

完成後，請確認 TradeArk 正在運行：

curl http://127.0.0.1:38182/health

如果你想從另一台裝置打開 TradeArk 頁面，請先在 Linux 防火牆或安全組中放行 TCP 38182 連接埠，然後訪問：

http://<server-ip>:38182/

By default, TradeArk uses port 38182 on Linux. The installer tries to set it to start automatically for the current user, and if you do not use root it installs under your user directory. Later you can change Web UI access from Settings > Web UI Access, or reset it from the shell with: tradeark web-ui set-password --username <name>

1
在本地安裝 TradeArk

請安裝在運行 AI 工具或交易工作流程的同一台機器上。

2
添加交易所 API 密鑰

填寫 API Key 和 Secret 即可。OKX 和 Bitget 還需要 passphrase。

3
交易前先檢查資料

在下真實訂單前，先確認餘額、持倉和訂單歷史都能正確讀取。

常見問題

這裡整理了大多數使用者開始前最常問的問題。

這會把 TradeArk 安裝到我的電腦上嗎？

會。安裝程式會把 TradeArk 下載到你的電腦上，並可讓它在背景本地運行，供你的 AI 工具或交易頁面使用。

我的交易所 API 密鑰會經過你們的伺服器嗎？

不會。你的 API 密鑰只保留在你自己的機器上。TradeArk 會在本地讀取它們，並由你的機器直接向交易所發出請求。

TradeArk 使用哪個連接埠？

預設使用 38182 連接埠。在同一台機器上打開 http://127.0.0.1:38182/ 。如果要從另一台 Linux 裝置訪問，請先在防火牆中放行該連接埠。

可以用密碼保護本地 Web UI 嗎？

可以。第一次開啟 TradeArk 時，可以選擇保持公開，也可以要求每次訪問都輸入使用者名稱和密碼。之後可在「設定 > Web UI 存取」中切換模式，或使用打包二進位裡的 web-ui set-password 命令重設密碼。

啟用 AI 交易時，怎樣做更安全？

先從 demo 模式開始，真實下單保留人工確認，並在讓 AI 發起真實交易前設定好槓桿和風險限制。
