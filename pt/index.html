AI Risk Control

Let AI trade through a local firewall, not through raw exchange keys

TradeArk includes a built-in two-layer AI trading firewall for OpenClaw, Claude Code, Codex, and similar local agents. Create a named AI client in the local Web UI, generate a ready-to-paste prompt, and keep TradeArk enforcing limits before any exchange write leaves the machine.

Per-agent token scope

Issue a separate AI client for each bot, workspace, or operator. Each token is bound to selected saved accounts and can be rotated or revoked instantly from the local Web UI.

Pre-trade policy engine

Before an order reaches the exchange, TradeArk can enforce route allowlists, symbol rules, asset scope, leverage ceilings, max quantity, max notional, and rate or daily limits.

Live watchdog + kill switches

A second layer keeps watching balances and positions from the local executor. It can alert, cancel open orders, flatten swap exposure, or freeze a client or account when thresholds are hit.

Prompt-ready onboarding

The AI Risk panel can generate a ready-to-paste prompt for OpenClaw, Claude Code, Codex, and similar tools, so teams do not need to handwrite headers, account scope, or route rules.

Normal local reads such as balances, positions, and history stay on the same machine, and exchange writes run on the same local trading routes under AI Risk Control by tagging each request with the named client.

Operators can rotate tokens in one click, freeze a client or account, and keep each AI workflow inside saved-account scope instead of handing out unrestricted live exchange keys.

AI Control Plane

How the local AI risk loop works end to end

The operator owns the control plane. AI gets prompt-ready access, TradeArk evaluates every protected write locally, and the exchange only sees requests that pass policy.

Operator -> AI prompt -> Local reads -> Protected writes -> Policy gate -> Exchange -> Watchdog
1
Create a named AI client

Bind it to saved accounts, allowed routes, and policy limits from the local Web UI.

2
Generate a prompt the AI can actually use

The prompt tells the AI which local routes to call and how to tag each request with the named client for protected writes.

3
Run exchange writes through the named local client

Orders, close-all, and cancel-open-orders hit the local firewall instead of bypassing it with raw exchange credentials.

4
Block, forward, or freeze

Layer 1 can block before the exchange sees the request. Layer 2 can alert, cancel, flatten, or freeze after exposure appears.

What the AI gets

A generated prompt lists the local routes to call and the named-client tag the executor uses to apply risk policy to protected writes.

What the operator keeps

One-click rotation, revoke, freeze, account separation, and a clear audit trail for every protected write decision.

O que é instalado na sua máquina

TradeArk is installed on the same machine where you trade. Setup downloads the program, creates local data files, and can enable automatic startup so your AI tool always connects to the same local service.

Detalhes da instalação

Se você está instalando, use o instalador padrão. Para preparar um servidor ou fluxo de trabalho de IA, use o comando de instalação em uma linha.

Windows

Portable ZIP for the local browser-based trading UI. On first open you can keep the page public or require a username and password every time. No admin rights, registry writes, or auto-start.

Download Windows portable ZIP Windows x64 · 1.0.0+20260615.074339Z

Extract the ZIP into any user-writable folder, then run the local launcher:

1. Download and extract tradeark_windows_portable.zip
2. Open the extracted folder
3. Run Start TradeArk.cmd
4. Your browser opens the local UI and the first-run access wizard

If you prefer a scripted install into LocalAppData, use the PowerShell helper:

Invoke-WebRequest -Uri https://tradeark.ai/install.ps1 -OutFile install.ps1
powershell -ExecutionPolicy Bypass -File .\install.ps1 -Yes

The local UI opens at:

http://127.0.0.1:38182/

This Windows package stays in user space. It does not require admin rights, does not write startup registry entries, and only notifies about new versions. Later you can change Web UI access from Settings > Web UI Access, or reset it from the portable folder with: TradeArk.exe web-ui set-password --username <name>

macOS

Menu bar app that installs TradeArk locally and lets you open the trading page or restart the local service with one click. On first open you can leave the page public or protect it with a username and password.

Download TradeArk for macOS (.dmg) Baixar DMG Intel macOS 13+ · Apple Silicon / Intel

Para a maioria dos usuários macOS desktop, use o app empacotado:

1. Download and open the TradeArk macOS package (.dmg)
2. Arraste o app baixado para /Applications
3. Abra o app e use o ícone da barra de menus para abrir a interface local no navegador
4. Use o menu para reiniciar o serviço local ou ativar o início em segundo plano

Este DMG pode ser aberto normalmente no Finder, mas ainda não tem notarização da Apple. Na primeira vez, o macOS pode pedir que você clique com o botão direito no app e escolha Abrir uma vez.

Se precisar de implantação por linha de comando ou configuração autônoma, ainda pode usar o comando de instalação em uma linha:

curl -fsSL https://tradeark.ai/install.sh | bash -s -- --non-interactive --skip-credentials --install-service

The command-line install sets up the same local TradeArk service. The app is simply a more convenient desktop way to manage it. If you enable password mode, every browser visit asks you to sign in until you switch back to public access or reset the password from Settings > Web UI Access.

Linux (Ubuntu / Debian)

Run one command and Linux will download TradeArk, install it the first time, or update the existing install in place later. On first open you can keep the page public or require a username and password every time:

curl -fsSL https://tradeark.ai/install.sh | bash -s -- --non-interactive --skip-credentials --install-service

When TradeArk is already installed, rerunning the same command keeps your local data, reuses the current Linux service settings when possible, and restarts the local service automatically.

Once the Linux build itself reports a new version inside the local Web UI, clicking the top-right update prompt now runs the in-place Linux update automatically and shows progress until the local service comes back. Windows and macOS still open manual download/update links.

After it finishes, confirm that TradeArk is running:

curl http://127.0.0.1:38182/health

If you want to open the TradeArk page from another device, allow TCP port 38182 on the Linux firewall or security group, then visit:

http://<server-ip>:38182/

By default, TradeArk uses port 38182 on Linux. The installer tries to set it to start automatically for the current user, and if you do not use root it installs under your user directory. Later you can change Web UI access from Settings > Web UI Access, or reset it from the shell with: tradeark web-ui set-password --username <name>

Perguntas frequentes

Respostas rápidas às perguntas mais comuns antes de começar.

Will this install TradeArk on my computer?

Yes. The installer downloads TradeArk to your computer and can keep it running locally in the background for your AI tool or trading page.

Minhas chaves API da exchange passam pelo seu servidor?

No. Your API keys stay on your own machine. TradeArk reads them locally and sends requests from your machine directly to the exchange.

Which port does TradeArk use?

By default, TradeArk uses port 38182. On the same machine, open http://127.0.0.1:38182/ . If you want to access it from another Linux device, open that port in the firewall first.

Can I protect the local Web UI with a password?

Yes. The first time you open TradeArk, it can stay public or require a username and password for every visit. Later you can switch modes from Settings > Web UI Access, or reset the password from the packaged binary with the web-ui set-password command.

Como me manter seguro ao ativar o trading com IA?

Comece com o modo demo, mantenha confirmação manual para ordens reais e defina limites de alavancagem e risco antes de permitir que a IA coloque operações reais.
