AI 风控防火墙

让 AI 通过本地防火墙交易，而不是直接持有原始交易所密钥

TradeArk 内置两层 AI 交易防火墙，面向 OpenClaw、Claude Code、Codex 等本地智能体。你可以在本地 Web UI 中创建具名 AI 客户端，生成可直接粘贴的提示词，并让 TradeArk 在任何交易所写请求离开本机前先执行限制。

按智能体划分令牌范围

为每个机器人、工作区或操作员单独创建 AI 客户端。每个令牌都绑定到选定的已保存账户，并可在本地 Web UI 中即时轮换或撤销。

交易前策略引擎

订单到达交易所前，TradeArk 可以执行路由白名单、交易对规则、资产范围、杠杆上限、最大数量、最大名义金额以及频率或每日限制。

实时看护与熔断开关

第二层会持续通过本地执行器监控余额和持仓。当触发阈值时，它可以告警、撤销挂单、平掉合约敞口，或冻结客户端/账户。

可直接用于提示词的接入流程

AI 风控面板可为 OpenClaw、Claude Code、Codex 等工具生成可直接粘贴的提示词，团队无需手写请求头、账户范围或路由规则。

余额、持仓、历史等普通本地读取都留在同一台机器上，而交易所写操作只需为每个请求加上已命名客户端标记，就能在 AI 风控下走相同的本地交易路由。

操作员可以一键轮换令牌、冻结客户端或账户，并把每个 AI 工作流限制在已保存账户范围内，而不是发放不受限制的真实交易所密钥。

AI 控制平面

本地 AI 风控闭环如何端到端工作

控制平面由操作员掌握。AI 获得可直接使用的提示词访问方式，TradeArk 在本地评估每个受保护写请求，交易所只会看到通过策略的请求。

操作员 -> AI prompt -> 本地读取 -> 受保护写入 -> 策略网关 -> Exchange -> 看护器
1
创建具名 AI 客户端

在本地 Web UI 中将它绑定到已保存账户、允许路由和策略限制。

2
生成 AI 可直接使用的提示词

提示词会告诉 AI 该调用哪些本地路由，以及如何为受保护的写操作给每个请求加上已命名客户端标记。

3
通过已命名的本地客户端执行交易所写操作

下单、全部平仓和撤销挂单都会先进入本地防火墙，而不是用原始交易所凭证绕过控制。

4
拦截、放行或冻结

第一层可在交易所看到请求前先拦截。第二层可在风险敞口出现后告警、撤单、平仓或冻结。

AI 获得什么

生成的提示词会列出要调用的本地路由，以及执行器用来对受保护写操作施加风控策略的已命名客户端标记。

操作员保留什么

一键轮换、撤销、冻结、账户隔离，以及每一次受保护写入决策的清晰审计记录。

你的机器上会安装哪些内容

TradeArk 会安装在你实际交易的那台机器上。安装流程会下载程序、创建本地数据文件，并可启用自动启动，让你的 AI 工具始终连接到同一个本地服务。

TradeArk 核心程序

这是安装在你电脑上的本地程序，提供交易页面、本地连接以及交易操作。

本地设置与记录

你的设置和其他本地数据都会保留在这台机器上，而不是存到远程网页账户里。

后台自动启动

安装完成后，TradeArk 可以在后台自动启动，你不必每次都手动重新打开。

安装说明

如果你是自己安装，请使用常规安装方式；如果你在准备服务器或 AI 工作流，请使用一行安装命令。

Windows

Portable ZIP for the local browser-based trading UI. On first open you can keep the page public or require a username and password every time. No admin rights, registry writes, or auto-start.

下载 Windows 便携 ZIP 包 Windows x64 · 1.0.0+20260701.104116Z

将 ZIP 解压到任意用户可写目录，然后运行本地启动器：

  1. 下载并解压 tradeark_windows_portable.zip
  2. 打开解压后的文件夹
  3. 运行 Start TradeArk.cmd
  4. Your browser opens the local UI and the first-run access wizard

如果你更喜欢用脚本安装到 LocalAppData，请使用这个 PowerShell 辅助脚本：

Invoke-WebRequest -Uri https://tradeark.ai/install.ps1 -OutFile "$env:TEMP\tradeark-install.ps1"; powershell -ExecutionPolicy Bypass -File "$env:TEMP\tradeark-install.ps1" -Yes

本地界面地址：

http://127.0.0.1:38182/

This Windows package stays in user space. It does not require admin rights, does not write startup registry entries, and only notifies about new versions. Later you can change Web UI access from Settings > Web UI Access, or reset it from the portable folder with: TradeArk.exe web-ui set-password --username <name>

macOS

Menu bar app that installs TradeArk locally and lets you open the trading page or restart the local service with one click. On first open you can leave the page public or protect it with a username and password.

下载 TradeArk for macOS (.dmg) 下载 Intel DMG macOS 13+ · Apple Silicon / Intel

对大多数 macOS 桌面用户来说，直接使用打包好的应用即可：

  1. 下载并打开 TradeArk macOS 安装包 (.dmg)
  2. 将下载的应用拖入 /Applications
  3. 打开应用，然后使用菜单栏图标在浏览器中打开本地界面
  4. 通过菜单重启本地服务，或打开后台自动启动

这个 DMG 可以在 Finder 中正常打开，但还没有完成 Apple notarization。第一次打开时，macOS 仍可能要求你右键应用并选择一次“打开”。

如果你需要命令行部署或无人值守安装，也可以继续使用一行安装命令：

curl -fsSL https://tradeark.ai/install.sh | bash -s -- --non-interactive --skip-credentials --install-service

The command-line install sets up the same local TradeArk service. The app is simply a more convenient desktop way to manage it. If you enable password mode, every browser visit asks you to sign in until you switch back to public access or reset the password from Settings > Web UI Access.

Linux (Ubuntu / Debian)

运行一条命令，Linux 会下载 TradeArk；首次运行会完成安装，之后可原地更新已有安装。第一次打开时，你可以选择保持页面公开，或每次访问都要求用户名和密码：

curl -fsSL https://tradeark.ai/install.sh | bash -s -- --non-interactive --skip-credentials --install-service

当 TradeArk 已安装时，再次运行同一命令会保留本地数据，尽可能复用当前 Linux 服务设置，并自动重启本地服务。

当 Linux 版本在本地 Web UI 中提示有新版本时，点击右上角更新提示会自动执行原地更新，并显示进度直到本地服务恢复。Windows 和 macOS 仍会打开手动下载/更新链接。

完成后，请确认 TradeArk 正在运行：

curl http://127.0.0.1:38182/health

如果你想从另一台设备打开 TradeArk 页面，请先在 Linux 防火墙或安全组中放行 TCP 38182 端口，然后访问：

http://<server-ip>:38182/

By default, TradeArk uses port 38182 on Linux. The installer tries to set it to start automatically for the current user, and if you do not use root it installs under your user directory. Later you can change Web UI access from Settings > Web UI Access, or reset it from the shell with: tradeark web-ui set-password --username <name>

Uninstall TradeArk

Use the uninstall helper when you want to remove the local service, installed binary, and registered local-agent skills. Local data is kept by default so account profiles and settings are not deleted accidentally.

Linux / macOS command-line uninstall:

curl -fsSL https://tradeark.ai/uninstall.sh | bash -s -- --non-interactive

Windows scripted uninstall:

Invoke-WebRequest -Uri https://tradeark.ai/uninstall.ps1 -OutFile uninstall.ps1
powershell -ExecutionPolicy Bypass -File .\uninstall.ps1 -Yes

To remove local data as well, add the explicit data-removal flag after you have verified that you no longer need saved settings or profiles:

# Linux / macOS
curl -fsSL https://tradeark.ai/uninstall.sh | bash -s -- --non-interactive --remove-data

# Windows
powershell -ExecutionPolicy Bypass -File .\uninstall.ps1 -Yes -RemoveData

The uninstall helper also removes TradeArk skill registrations from Codex, OpenClaw, OpenCode, Claude, and generic local-agent skill folders when those folders exist.

1

请安装在运行 AI 工具或交易工作流的同一台机器上。

2
添加交易所 API 密钥

3
交易前先检查数据

常见问题

这里汇总了大多数用户开始前最常问的问题。

这会把 TradeArk 安装到我的电脑上吗？

会。安装程序会把 TradeArk 下载到你的电脑上，并可让它在后台本地运行，供你的 AI 工具或交易页面使用。

我的交易所 API 密钥会经过你们的服务器吗？

不会。你的 API 密钥只保留在你自己的机器上。TradeArk 会在本地读取它们，并由你的机器直接向交易所发出请求。

TradeArk 使用哪个端口？

默认使用 38182 端口。在同一台机器上打开 http://127.0.0.1:38182/ 。如果要从另一台 Linux 设备访问，请先在防火墙中放行该端口。

可以用密码保护本地 Web UI 吗？

可以。第一次打开 TradeArk 时，可以选择保持公开，也可以要求每次访问都输入用户名和密码。之后可在“设置 > Web UI 访问”中切换模式，或使用打包二进制里的 web-ui set-password 命令重置密码。

启用 AI 交易时，怎样做更安全？

先从 demo 模式开始，真实下单保留人工确认，并在让 AI 发起真实交易前设置好杠杆和风险限制。
